# Cafe API (Go)

Небольшой HTTP-сервис на Go, который возвращает список кафе по городу  
с поддержкой фильтрации, поиска и ограничения количества результатов.

Проект реализован как **pet-backend**, с упором на чистую архитектуру,
тестируемость и читаемость кода.

---

## Возможности

- получение списка кафе по городу
- фильтрация по подстроке в названии
- ограничение количества результатов
- JSON API
- unit-тесты бизнес-логики
- тесты HTTP-handler’ов
- CI с проверкой тестового покрытия

---

## Пример запроса

```http
GET /cafes?city=moscow&search=кофе&count=2
```

### Пример ответа

```json
[
  "Мир кофе",
  "Кофе и завтраки"
]
```

---

## Архитектура

Проект разделён на слои по ответственности:

```
cmd/server        — точка входа, запуск HTTP-сервера
internal/handler  — HTTP-handlers (парсинг запроса, формирование ответа)
internal/service  — бизнес-логика
internal/model    — структуры доменной модели
data              — источник данных (in-memory)
```

Такой подход:
- изолирует бизнес-логику от HTTP
- упрощает тестирование
- позволяет легко заменить источник данных (например, на БД)

---

## Тестирование

Тестируются два уровня:

- **service** — чистая бизнес-логика
- **handler** — HTTP-контракт (через `httptest`)

Запуск всех тестов:

```bash
go test ./...
```

---

## Запуск проекта локально

```bash
git clone https://github.com/MaximK0valev/cafe-api-go.git
cd cafe-api-go

go mod tidy
go run ./cmd/server
```

Сервер будет доступен по адресу:

```
http://localhost:8080/cafes
```
